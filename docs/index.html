Architettura dell’applicazione

L’applicazione è strutturata secondo una separazione chiara delle responsabilità, ispirata al pattern MVC, pur senza l’utilizzo di framework avanzati.

Dominio

Contiene la logica di business dell’applicazione: gestione ordini, carrello, prodotti e stati. Non conosce HTTP né database.

Esempi: Ordine, Prodotto, Carrello, GestoreOrdini
Database

Gestisce l’accesso ai dati tramite JDBC. Le query SQL sono centralizzate e le connessioni vengono fornite in modo stateless.

Esempi: GestoreDatabase, Query
Server (HTTP)

Gestisce le richieste HTTP tramite HttpServer Java. Ogni rotta è associata a un HttpHandler dedicato.

Esempi: MenuHandler, OrdineHandler, DashboardStaffHandler
Presentazione (HTML)

Contiene i template HTML statici, separati dalla logica. Le parti dinamiche vengono inserite dagli handler.

Esempi: MenuHtml, CarrelloHtml, DashboardStaffHtml

Il progetto è sviluppato senza framework web (Spring, Hibernate) per comprendere in profondità il funzionamento di HTTP, JDBC e della gestione dello stato applicativo.
